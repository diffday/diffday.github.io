高效团队

  • 软件开发迭代周期和频率越来越快
  • 管理问题
    • 团队越来越大，分布在不同区域
    • 执行力问题，流动问题，成长问题

不是所有公司都需要高效团队，但当处理的业务复杂度，交互复杂度，或服务用户数越来越多，就需要高效团队了。要又快又好，还要降低成本，如上提及的管理问题也在凸显。

如何搞成高效团队

  • 搞敏捷
  • 用权威，干不完不用下班
  • 使命感价值观，要不停地努力，这是对你好
  • KPI，996，钉钉 等等

如上保证不了你的公司你的团队去更高的位置，解决更复杂更难的问题，**有更有价值的产出，终极的问题还是在团队的人上，再看团队中的事，再之团队领导力**。

女性婚恋相亲窘境

发表于 2021-08-13 分类于 网文精摘
本文字数： 949 阅读时长 ≈ 1 分钟

“割韭菜”的相亲平台

标题党，但又折射出相亲的矛盾。相亲的本质就是一帮最不会给女人提供情绪价值的男人遇见的了一帮最需要情绪价值的女人，再真实一点，就是一帮最不会追女人的男人遇到了一帮最难追的女人，再再真实一点，就是一帮要求很高的男人遇到了一帮要求也很高的女人，他们能不互相嫌弃吗？

现实世界满足不了的幻想，相亲平台来满足你。现实中找不到理想的另一半，相亲平台都有。实际上稍微有点吸引力的人，身边的追求者都应付不过来，哪有时间精力来相亲平台登记呢？

在婚恋市场上，一旦你进步了相亲这个环节，虽然表面看上去你有很多的选择权，但实际意味着，你其实没有选择权了。

男女方的认知偏差

男人想的是我越相亲我越不想付出，但是女人的想法恰恰相反，我越相亲我越要找一个能对我付出的，否则我等了35年白等了。反正沉默成本这么高，大不了再等三年呗，但是再三年，她的要求就更高了。但男人想的是，又过了三年，价格该打折了。所以这些男人和女人之间的需求，完全是错配的，且女性更难一些。

女性的特质囧境分析

单身人士如果相亲不能遇到真爱，怎么才能遇到真爱呢，其实这个问题关键因素有两个。

不知道对方什么需求

一是不知道对方什么需求，比如很多女生以为男人喜欢的特点是女生读世界名著，钢琴，瑜伽，对小动物有爱心，环保做义工，有份好工作，吃有机食品不吃烤串，其实这些男人都不在乎。男生在乎的无非是温柔体贴加好看。

如果把相亲的人比做滞销产品，还没找到另一半真正的原因，是你自认为的优点，根本不符合客户的需求。如果一个产品卖不出去，滞销了怎么办，这个时候，你不应该埋头生产更多不符合市场需求的产品，各种客户不需要的功能都加上去，越卖越贵。比如弹个钢琴，做个瑜伽，读书旅游，然后强行推销。而是应该立即停产，要么降价大促销，要么调整配方再出售。相亲也是一样的道理。

高估自己

其次，人都会高估自己。比如女生会高估自己的容貌，直男行规是平方/10。比如有位女性宣称自己有8分，直男心算6.4分，女生宣称自己6分中等美，实际上是3.6分。1957年美国一个实验也能证实，你觉得你的智商是否超过了社会平均？91%的美国人选择了是，反映到相亲市场也是一样的，90%的人认为自己在平均线以上，所以要求对方也要在平均线以上。

结语

所以了解对方的需求，合理评估自己，才能找到真爱。

游戏发展史

美国篇

美国游戏历史发展连贯，自1947年第一个电子游戏（阴极射线管娱乐装置）诞生，第一个主机游戏（1967 Brown Box），电脑游戏（1962，太空战争），第一个网游，第一个手游（1994 俄罗斯方块，贪吃蛇还要靠后），多类游戏均诞生在美国。期间经历过家用游戏平台重量不重质的雅达利崩溃，让位让日本游戏主机厂商有了居上的契机。总体上美国游戏产业始终健康发展，形成主机，端游，手游三足鼎立局面，也在主要的主机游戏中形成任天堂，索尼及微软三足鼎立的稳定态势。

苹果两位创始人都曾是雅达利的游戏开发工程师，他们在1974年一同合作，仅用4天就完成了一个《打砖块》的雏形游戏–《突出重围》– 关卡升级玩法机制的越狱游戏，首次加入了斜角度物体运动概念，反弹两侧墙壁造成连锁撞击效果

人脑云计算：没大数据时如何预测

土法炼钢-德菲尔法

把专家的独立观点不断收敛

  1. 从各领域邀请20位权威专家，预测最高，最低，最可能三个值
  2. 独立预测，不要召专家在一起开会讨论，避免相互影响
  3. 统计回归：归纳整理后，匿名反馈给各位专家，然后请专家参考修正自己的预测。如此往复几次，一般到第四次预测时，大部分专家已不再修改自己的意见，彼此的预测也越来越接近（收敛）
  4. 分析结果，用“主观概率加权平均法”，最低25%权重，最高25%权重，最可能50%权重，加权求和，得到你想要的预测值。

在20世纪中期，此套方法就在发生功效，准确预测朝鲜战争的结果。此套方法可充分利用专家的专业判断，在一些很难定量分析预测的问题上，实现“人脑云计算”，获得相对准确的预测结果。使用的注意事项是专家可以多样化，专家不一定是咨询公司，也是可以是一线管理人员，可以是客户，但他们必须独立给出预测。
